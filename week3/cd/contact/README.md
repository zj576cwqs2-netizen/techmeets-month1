# contact.php - お問い合わせフォーム

## 概要

PHPのみで実装した3ステップのお問い合わせフォームです。  
外部ライブラリは不要で、単一ファイル（`contact.php`）で完結しています。

---

## 作成したフォームの内容

- 入力項目：名前・メールアドレス・件名・メッセージ
- バリデーション：全項目必須、メールアドレスの形式チェック
- XSS対策：`htmlspecialchars` によるエスケープ処理
- 3ステップ構成：入力 → 確認 → 完了

---

## 工夫した点

内容の確認として、回答内容がメールアドレスに送信されるように工夫した。

---

## 学んだこと

AIの書くコードには限界があるから、AIと人間が良いバランスを保ちコードを書く必要があると感じました。

---

## 画面の流れ

```
[ステップ1] 入力フォーム
      ↓ 「確認画面へ →」ボタン（バリデーション実行）
[ステップ2] 確認画面（入力内容を一覧表示）
      ↓ 「送信する」ボタン          ← 「入力に戻る」で修正可能
[ステップ3] 完了画面（送信内容を表示 + 確認メール送信）
```

---

## 入力項目

| フィールド名 | 種別 | 必須 |
|---|---|:---:|
| 名前 | テキスト | ○ |
| メールアドレス | メール | ○ |
| 件名 | テキスト | ○ |
| メッセージ | テキストエリア | ○ |

---

## 機能

### バリデーション（ステップ1 → 2）

- 全項目の空欄チェック
- メールアドレスの形式チェック（`filter_var` + `FILTER_VALIDATE_EMAIL`）
- エラーがある場合は入力値を保持したままエラーメッセージを表示

### XSS 対策

すべての出力箇所で `htmlspecialchars($value, ENT_QUOTES, 'UTF-8')` を使用しています。  
`ENT_QUOTES` によりシングル・ダブルクォートの両方をエスケープするため、  
HTML属性（`value="..."`）への埋め込みも安全です。

### 確認メールの送信（ステップ2 → 3）

「送信する」ボタンを押すと、入力されたメールアドレス宛に確認メールを送信します。

```
件名: 【お問い合わせ受付】{件名}
宛先: 入力されたメールアドレス
送信元: no-reply@example.com
```

本文には名前・メールアドレス・件名・メッセージが記載されます。

送信には `mb_send_mail()` を使用しており、日本語の件名・本文を正しく扱うために  
`mb_language('Japanese')` と `mb_internal_encoding('UTF-8')` を設定しています。

完了画面では送信結果（成功 / 失敗）をメッセージで通知します。

---

## ステップ管理の仕組み

画面遷移は hidden フィールド `name="step"` の値で制御しています。

| `step` の値 | 意味 |
|---|---|
| `form` | ステップ1の入力フォームから送信された |
| `confirm` | ステップ2の確認画面から送信された |

「入力に戻る」ボタンは hidden フィールドで全入力値を保持して POST するため、  
入力内容がリセットされません。

---

## ローカル環境での起動方法

PHP の組み込みサーバーを使って動作確認できます。

```bash
php -S localhost:8000 -t /path/to/php-practice
```

ブラウザで `http://localhost:8000/contact.php` を開いてください。

> **注意**: Windows のローカル環境では `mail()` / `mb_send_mail()` はデフォルトで動作しません。  
> メール送信を有効にするには `php.ini` の SMTP 設定、または本番サーバーへのアップロードが必要です。
